### Неоднозначность

**Ограничение времени:** 1 с

**Ограничение памяти:** 512.0 Мб

**Ввод:** стандартный ввод или input.txt

**Вывод:** стандартный вывод или output.txt

Пусть $b$ — основание системы счисления. Обычно для записи числа в такой системе используются первые $b$ символов из последовательности
`0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ` .
Значения цифр совпадают с их классическим пониманием, а затем идут буквы: $A = 10$, $B = 11$, …, $Z = 35$.

Запись из $k$ символов $d_{k-1} \dots d_1 d_0$ в системе счисления с основанием $b$ соответствует числу $d_0 + d_1 \cdot b + \dots + d_{k-1} \cdot b^{k-1}$.

Обычно каждое целое число в такой системе счисления можно записать единственным способом.
Но что если для записи доступны не все символы? Или, наоборот, доступны символы, которые обычно недоступны?

По заданному числу $n$ вам нужно определить, однозначна ли его запись в системе счисления с основанием $b$, если для записи доступно только некоторое заданное множество символов. Запись числа не может начинаться с нуля.

Например, при $b = 2$ следующие записи соответствуют числу $25$: `P` , `B3` , `11001` .

### Формат ввода

Первая строка входных данных содержит одно целое число $b$ $(2 \leq b \leq 36)$.

Вторая строка содержит множество доступных для записи символов. Каждый символ — цифра от $0$ до $9$ или заглавная латинская буква от $A$ до $Z$.
Символы заданы в порядке возрастания своих значений. Гарантируется, что среди них есть символ $0$.

Третья строка содержит одно целое число $n$ $(1 \leq n \leq 10^{100})$, записанное в десятичной системе счисления.

### Формат вывода

Если не существует ни одной записи числа $n$ с помощью данного множества символов, выведите одну строку:

```
Impossible
```

Если существует ровно один способ записи числа $n$, в первой строке выведите:

```
Unique
```

а во второй строке — искомую запись числа $n$.

Если существует по крайней мере два способа записи числа $n$, в первой строке выведите:

```
Ambiguous
```

а во второй и третьей строках — любые две различные записи числа $n$.

**Пример 1**

**Ввод**

```
10
0123456789A
110
```

**Вывод**

```
Ambiguous
110
AA
```

**Пример 2**

**Ввод**

```
10
023456789
10
```

**Вывод**

```
Impossible
```

**Пример 3**

**Ввод**

```
10
023456789A
10
```

**Вывод**

```
Unique
A
```

**Примечание:** в первом тестовом примере существует ещё одна допустимая запись числа $110$ — строка `10A` . Однако выводить все возможные представления не требуется, достаточно любых двух различных. Решение представлено в файле 'C.cpp` .
